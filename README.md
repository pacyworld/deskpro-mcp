# Deskpro MCP Server

A PHP [MCP](https://modelcontextprotocol.io/) (Model Context Protocol) server for [Deskpro](https://www.deskpro.com/) helpdesk, built on the [Enchilada Framework](https://buenapp.org/docs/).

Provides ticket management, search, and agent tools via the MCP stdio transport - including OAuth Bearer token authentication with automatic refresh, which is not available through the standard API key method alone.

## Features

- **22 tools** for complete helpdesk management from an AI assistant
- **Ticket operations** - get, create, update, search, list with pagination and filters
- **Agent actions** - reply, internal note, assign, change status, view messages
- **Lookups** - departments, agents, statuses, custom fields, saved filters
- **Self-documenting** - built-in help, tool categories, usage examples, error guide
- **Dual authentication** - API Key or OAuth Bearer token with automatic refresh
- **Multi-instance** - manage multiple Deskpro helpdesks from a single server
- **Zero dependencies** - no Composer, no npm; just PHP 8.2+ with `curl` and `json`
- **PHAR distribution** - single-file deployment (~97 KB)

## Requirements

- PHP 8.2 or later
- PHP extensions: `curl`, `json`, `phar` (for PHAR distribution)
- A Deskpro Cloud or On-Premise instance with one of:
  - An API key (generated by a Deskpro admin), or
  - OAuth Bearer tokens (from the Deskpro web app session)

## Quick Start

### PHAR (recommended)

Download the latest PHAR from [Releases](https://pacyworld.dev/pacyworld/deskpro-mcp/releases):

```bash
curl -LO https://pacyworld.dev/pacyworld/deskpro-mcp/releases/latest/download/deskpro-mcp.phar
chmod +x deskpro-mcp.phar
```

Create a config file:

```bash
mkdir -p ~/.config/deskpro-mcp
cat > ~/.config/deskpro-mcp/deskpro.json << 'EOF'
{
    "auth_method": "apikey",
    "api_token": "YOUR_API_KEY",
    "site_url": "https://yourcompany.deskpro.com"
}
EOF
```

Start the server:

```bash
php deskpro-mcp.phar
```

### From Source

```bash
git clone https://pacyworld.dev/pacyworld/deskpro-mcp.git
cd deskpro-mcp
cp config/deskpro.json.sample config/deskpro.json
# Edit config/deskpro.json with your credentials
php bin/deskpro-mcp
```

## Available Tools

| Tool | Description |
|------|-------------|
| `deskpro_get_ticket` | Get ticket details by ID |
| `deskpro_create_ticket` | Create a new ticket |
| `deskpro_update_ticket` | Update ticket fields |
| `deskpro_search_tickets` | Search tickets by keyword, status, agent, department |
| `deskpro_list_tickets` | List tickets with pagination and filters |
| `deskpro_add_reply` | Reply to a ticket (visible to user) |
| `deskpro_add_note` | Add an internal agent note |
| `deskpro_get_messages` | Get messages/replies for a ticket |
| `deskpro_assign_ticket` | Assign ticket to an agent |
| `deskpro_change_status` | Change ticket status |
| `deskpro_list_departments` | List available departments |
| `deskpro_list_agents` | List agents (support staff) |
| `deskpro_list_statuses` | List ticket statuses |
| `deskpro_get_ticket_fields` | Get custom field definitions |
| `deskpro_list_filters` | List saved ticket filters/views |
| `deskpro_list_instances` | List configured Deskpro instances |
| `deskpro_switch_instance` | Switch the active default instance |
| `deskpro_help` | Context-aware help for AI agents |
| `deskpro_tool_categories` | List tool categories with descriptions |
| `deskpro_usage_examples` | Common workflow examples |
| `deskpro_server_info` | Server version, capabilities, configured instances |
| `deskpro_error_guide` | Common errors and resolution steps |

All ticket, action, and lookup tools accept an optional `instance` parameter to target a specific Deskpro site.

## Authentication

Two methods are supported. You only need one.

### API Key (recommended for stability)

Requires a Deskpro admin to generate a key under Admin > Apps & Integrations > API Keys.

```json
{
    "auth_method": "apikey",
    "api_token": "YOUR_API_KEY",
    "site_url": "https://yourcompany.deskpro.com"
}
```

### OAuth Bearer Token

Uses JWT tokens from the Deskpro web app session. No admin access needed. Tokens auto-refresh every 3 hours; the server writes refreshed tokens back to the config file.

```json
{
    "auth_method": "token",
    "access_token": "eyJ...",
    "refresh_token": "def502...",
    "site_url": "https://yourcompany.deskpro.com"
}
```

Get tokens from your browser cookies: `app_access_token` and `app_refresh_token` (Application > Cookies in developer tools).

### Multi-Instance

Manage multiple Deskpro helpdesks from a single server:

```json
{
    "default": "production",
    "instances": {
        "production": {
            "auth_method": "apikey",
            "api_token": "PROD_KEY",
            "site_url": "https://support.example.com"
        },
        "staging": {
            "auth_method": "token",
            "access_token": "eyJ...",
            "refresh_token": "def502...",
            "site_url": "https://staging.deskpro.com"
        }
    }
}
```

The flat (single-instance) config format shown in Quick Start is also fully supported - the instance name is derived from the site URL automatically.

## IDE Integration

### Windsurf

Add to your MCP configuration:

```json
{
    "deskpro": {
        "command": "php",
        "args": ["/path/to/deskpro-mcp.phar"]
    }
}
```

With explicit config path:

```json
{
    "deskpro": {
        "command": "php",
        "args": ["/path/to/deskpro-mcp.phar", "--config=/path/to/deskpro.json"]
    }
}
```

### Other MCP Clients

Any MCP client supporting the stdio transport can use this server. Configure it to spawn `php /path/to/deskpro-mcp.phar` and communicate over stdin/stdout using JSON-RPC 2.0.

## Configuration Locations

The server searches for `deskpro.json` in this order (first match wins):

1. `DESKPRO_CONFIG` environment variable
2. `--config=/path/to/deskpro.json` CLI argument
3. `config/deskpro.json` (relative to server root)
4. `~/.config/deskpro-mcp/deskpro.json`
5. `/usr/local/etc/deskpro-mcp/deskpro.json`

## Building from Source

Build the PHAR distribution:

```bash
php -d phar.readonly=0 bin/build-phar.php
```

This produces `deskpro-mcp.phar` (~97 KB) in the project root.
